# 今後のモデレーション課題メモ

このドキュメントは、現時点では未実装の検討アイデアを残すためのメモです。  
実装方針はまだ固定せず、必要になったときに安全寄りで再検討する前提にしています。

## 現状の前提

- 2行の投稿は全文をまとめて判定している。
- 連絡先、SNS ID、URL、個人情報、強い表現、検索避け、悪戯や下品な表現は手動確認寄りに倒している。
- AI判定は OpenAI Moderation API とローカルの注意ルールを併用している。

## 今後の課題候補

- 縦読みの検出
- 行頭文字や行末文字を使った隠しメッセージの検出
- 絵文字だけで意味を隠す投稿の扱い
- 絵文字が多い短文を慎重判定に寄せるルール
- 記号や改行を混ぜた新しい検索避けへの追従
- ひらがな、カタカナ、半角カナ、英字、数字の混在パターンの精査
- 文字数は短いが、文脈的に注意が必要な投稿の扱い

## 検討の方向性

- まずは自動承認ではなく保留に倒す。
- 誤検知を少なくしたい項目と、見逃しを少なくしたい項目を分けて考える。
- ルールを増やしすぎる前に、実際の運用で出た例を記録してから調整する。

## メモ

- これは実装済みの仕様ではない。
- 追加したいときは、まず運用ログや実際の投稿例を見てから優先度を決める。
