# 次セッション引き継ぎメモ

対象リリース: `1.6.2`

## 現状

- 管理画面はサーバ共有の本設定、月メニューは投影端末ごとの現場微調整として整理済み。
- 管理画面と月メニューで調整できる項目の比較は `docs/OPERATIONS.md` の「管理画面と月メニューの調整項目比較」を参照。
- 月メニューは起動時に `body` 直下へ移動し、投影ステージ外の独立DOMとして扱う。全画面表示は `document.documentElement` を対象にする。
- 月メニュー表示中は、3Dボックス更新を止め、星空/天の川/流星Canvasを一時停止し、見える部分の少ない短冊描画を省略する。閉じたら復帰する。
- 月メニューのローカル調整は、操作した項目だけサーバ設定より優先する。保存値クリアでサーバ設定追従へ戻る。

## 重要な設計判断

- 短冊配置のサーバ保存は採用していない。短冊配置は投影端末ブラウザの `localStorage`。
- 3Dボックス位置とカメラ注視点は、管理キーが同じブラウザに保存されている場合のみ月メニュー操作からサーバへも保存する。
- 短冊奥行は画面上Y位置を使わず、z順だけから作る。
- 月メニューの描画安定化は、z-indexだけでなく、独立DOM化、背景描画停止、3Dボックス更新停止、Canvas停止の複合対策で行っている。
- 見える部分の少ない短冊の描画省略は通常時には体感効果が薄かったため、月メニュー表示中だけ有効。

## 次回候補

- Android実機で月メニューのチラつきが残るか確認する。残る場合は、月メニューをさらに軽量な不透明背景にし、半透明/影/スクロール領域の描画負荷を減らす。
- 管理画面の雲スライダーと短冊目標スライダーが広い画面でグループからはみ出る問題を確認し、グリッドの最小幅と折り返しを調整する。
- OSC経由で演出ON、プリセット1〜3読み込み、笹舟OSCモードを操作する案が未実装。
- 表示短冊数が多い時の処理落ちは、通常時の短冊軽量モードよりも、視差更新fps間引き、影/フィルタ削減、入替中の深度更新固定のほうが効果が出る可能性が高い。
