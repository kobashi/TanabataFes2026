# Development Notes

このドキュメントは、試作開発中に調整した判断や今後の開発で参照したい要点をまとめたものです。

## 全体方針

- 大学構内WiFiからのローカル運用を前提に、Node.js標準ライブラリだけで構成している。
- 外部DBは使わず、願い事は `data/wishes.json` に保存する。
- 管理者が承認した願い事だけを投影画面に表示する。
- 投影画面はプロジェクションマッピング用途を優先し、暗い背景、強い発色、縦書き短冊、星空・雲・月・流れ星の演出で構成する。

## 投稿入力

- 願い事は最大2行。
- 1行ごとにテキストボックスを分ける。
- 改行コードは無視し、貼り付け時も1行内に収める。
- 1行の最大文字数は13文字。
- Enterキーで投稿されないように、入力欄の `keydown` で送信を抑止している。
- 投稿画面のプレビューは投影画面と同じ `.tanzaku-text` を使い、文字配置の差が出にくいようにしている。

## 管理画面

- 管理キーは `ADMIN_KEY` 環境変数で指定する。未指定時は試作用の `tanabata2026`。
- 新規投稿や承認状態の変更は Server-Sent Events で管理画面へ通知する。
- SSEが不安定な場合に備え、30秒間隔のポーリングも残している。
- 投影画面にも Server-Sent Events を追加し、願い事、投影設定、演出更新をポーリング待ちなしで反映する。投影側にも既存ポーリングをフォールバックとして残す。
- 投影画面のポーリングでは、投影設定と承認済み願い事のシグネチャを比較し、差分がない場合は短冊再描画、3D視差refresh、雲DOM再生成を行わない。雲はCSS animationが進行中のDOMを維持し、設定変更時だけ `renderCloudLayer()` で作り直す。
- 管理キーはブラウザの `localStorage` に保存しているため、共有PCや公開端末では注意する。
- 開発・動作確認では本番用 `ADMIN_KEY` を使わず、孤立モード用の開発テスト管理キーを使う。本番キーはチャット、ターミナル出力、ログに残る形で扱わない。
- 孤立モードの管理キーは、起動時に `ADMIN_KEY=開発テスト用キー npm run dev:isolated` の形で明示する。
- 投影設定フォームは、入力中にSSEや30秒ポーリングの `refresh()` が走っても未適用の値をサーバー値で上書きしない。適用成功時だけ確定値で再描画する。
- 視差3D関連だけは現場で見ながら調整するため、300msのデバウンス付き即時プレビューにする。通常の投影設定は誤操作防止のため「適用」ボタンで確定する。
- 投影設定には3個のプリセット枠を用意する。保存済み/未保存が管理画面で分かるようにし、保存・読込・クリアを個別に操作する。
- 管理画面は「接続」「願い事管理」「投影設定」「バックアップ」のタブで分ける。現場用に承認待ち件数、投影中件数、注意つき件数などの概況と、承認待ちの一括投影/一括非表示ボタンを置く。

## バックアップ

- 願い事バックアップは `data/backups/wishes-<revision>.json` に保存する。
- `revision` はISO日時をファイル名向けにした値で、管理画面ではリビジョンとして扱う。
- 管理画面から手動バックアップを作成できる。
- レストア時は、現在の `wishes.json` を `before-restore` として自動バックアップしてから、指定リビジョンの願い事で置き換える。
- 既存の配列だけのバックアップファイルも読み込めるようにし、管理画面からのバックアップはメタ情報と `wishes` を含むJSONオブジェクトにする。

## 短冊表示

- 総スロット数、表示数、入れ替え数は管理画面から調整できる。既定は総スロット15個、表示12枚。
- 表示数は総スロット数を超えない。総スロット数を下げて表示数の上限が下がる場合は、表示数も上限内に調整する。
- 運用中に表示数を増やした場合は、追加表示されたスロットだけ空き場所探索を強制し、保存済みレイアウトの既存表示スロットは維持する。
- 入れ替え数は表示数を超えない。表示数を下げて入れ替え数の上限が下がる場合は、入れ替え数も上限内に調整する。
- 待機スロット数は `総スロット数 - 表示数` で決まる。
- 投影画面は `/api/approved` を7秒間隔で取得し、新規承認された願い事を検出する。
- 新規承認された願い事は通常ローテーションを待たず、表示中の一番古い短冊を待機へ戻して優先表示する。
- 通常ローテーションでは、表示中の一番古い短冊を待機へ戻し、待機中の一番古い短冊を表示へ上げる。
- 各スロットは `rotationOrder` を持ち、表示または待機に入った順番で古いものから入れ替える。
- 新規表示された短冊は一番新しい表示扱いになるため、次のローテーションですぐ消えにくい。
- 新規承認や通常ローテーションで表示へ上がる短冊は、既存短冊より前面に表示する。
- 投影端末に保存済みレイアウトがないスロットでは、新規短冊の表示時に候補位置から既存短冊との重なりが少ない場所を選ぶ。候補位置は総スロット数に応じて増やす。
- 投影画面の月は隠しメニューになっており、現在の短冊配置、短冊フォント、短冊フォントサイズ、笹舟サイズを投影端末の `localStorage` に1件だけ保存・呼び出しできる。
- 表示OKな願い事数が総スロット数より少ない場合でも、同じ願い事が表示中に複数出ないようにする。待機スロットが空なら、古い表示短冊を別スロットへ順次移して見せる。
- 入れ替え数は表示数以下に制限する。
- ローテーション中と新規取得が重なる稀なケースは、運用上の影響が小さいため許容する。
- 待機中のメッセージも保持し、表示数を超えた願い事はページングではなく周期的な入れ替えで見せる。
- 新規表示は一文字ずつ出るタイプ風演出。
- タイプ風演出は文字数に応じて速度を変え、短いメッセージほど加速度的に遅くする。
- 消える短冊は風に舞い上がって天に昇る演出。
- 表示中の短冊はドラッグで位置調整でき、位置は `localStorage` に保存される。
- 次回修正案: 表示短冊数が増えた時に、処理落ちで短冊が部分的に欠落したり表示が遅延する場合がある。まずは表示枚数が多い時の軽量モードを検討する。候補は、短冊の `box-shadow` と `filter` の軽量化、視差更新を30fpsまたは15fpsへ間引くこと、3DBox更新頻度のさらなる低下、入れ替え中の深度配列固定強化。
- 次回修正案: 縦書きテキストのCanvas/画像化は効果が大きい可能性があるが、フォント、絵文字、タイプ風演出、表示更新の複雑さが増えるため後段の切り札とする。DOM短冊の仮想化は投影画面では不向きなので、短冊を消すのではなく、表示短冊16枚以上などを目安に自動で軽く描く方向を優先する。

## 投影演出

- 管理画面の「流星群」ボタンから、投影画面へ短時間の演出リクエストを送る。
- 管理画面の投影設定で自動演出をONにすると、指定したイベント間隔で流星群演出を自動発生させる。
- 自動演出は既定OFF、イベント間隔の既定は `300000ms`。
- 演出リクエストはサーバーメモリ上の一時状態で、30秒で失効する。投稿データや設定ファイルには保存しない。
- 投影画面は演出リクエストを2.5秒間隔で確認する。
- 流星群レイヤーは月、笹、短冊より後ろに表示する。
- 流星群演出中は通常ローテーションと新規短冊の導入を一時停止する。
- 流星群の発生後、少し待ってから表示中の短冊を古い順に退場させる。
- 全短冊が退場した後、流星群をフェードアウトし、承認済み一覧を再反映して通常表示へ戻す。
- 投影演出アップグレードでは、月、星空、天の川は固定遠景として扱い、視差演出の対象外にする。
- 視差と奥行きの対象は、短冊と笹に限定する。短冊は表示中短冊の `z` 順だけから疑似深度を作り、笹は中景として扱う。短冊の画面上Y位置は投影面上の高さであり、奥行きに混ぜない。
- WebGL導入前の第一段階として、Canvas 2Dの演出レイヤーを既存DOMの背後に追加し、天の川、強化流星群、短冊の光まといを重ねる。
- 短冊の奥行きは表示中短冊を `z` 順に並べ、奥から手前へ均等に `0〜1` の疑似深度を割り当てる。手前の短冊ほどわずかに大きく明るく見せる。
- 笹は短冊より手前の中景として、左右でゆっくり揺らし、固定遠景との差で立体感を出す。
- 風の調整は `projectionTanzakuSwayStrength`、`projectionWindGustStrength`、`projectionWindGustCycleMs` で行う。短冊揺れ強度は回転寄り、突風強度は押し流し寄り、突風周期は再発生間隔として扱う。
- 天の川演出は管理画面の投影設定から、明るさ `projectionMilkyWayGain`、きらめき `projectionMilkyWayTwinkle`、強い瞬き `projectionMilkyWaySparkle`、変化速度 `projectionMilkyWaySpeed`、粒子数 `projectionMilkyWayParticleCount`、強い瞬き割合 `projectionMilkyWaySparkleRatio`、瞬き周期分散 `projectionMilkyWaySparklePeriodVariance`、瞬き強度分散 `projectionMilkyWaySparkleIntensityVariance` として調整する。既定は投影向けに明るさ `1.75`、粒子数 `340`、強い瞬き割合 `0.045`、その他 `1`。
- ベガとアルタイルは、通常の短い瞬きに加えて、互いに少し位相をずらした長周期の明滅を重ね、呼応しているように見せる。呼応の発光強度、位相差、周期は `projectionTanabataStarResponseIntensity`、`projectionTanabataStarResponsePhaseDeg`、`projectionTanabataStarResponsePeriodSeconds` として管理画面から調整する。
- 実験的な視差立体感は `projectionExperimentalParallaxEnabled` でON/OFFする。
- 笹舟の移動モードは `projectionParallaxMotionMode` で管理する。`mapping` は投影面に対して地面上の鑑賞ポイントが左右・前後に動く従来モード、`camera` は現実カメラも仮想カメラ移動に追従して撮影し、投影面が画角内でズームイン/アウトするぶんを相殺する左右・前後モード、`camera-display` は同じく実カメラ想定で左右・上下へ平行移動するモード、`display` は投影面と平行に左右・上下へ動くモードとして扱う。
- 左右移動倍率は `projectionParallaxStrength`、前後/上下移動倍率は `projectionParallaxPopoutStrength`、短冊奥行倍率は `projectionParallaxDepthMultiplier`、奥行基準順は `projectionParallaxDepthReferenceIndex`、鑑賞地点の左右オフセットは `projectionParallaxViewerOffsetX`、上下オフセットは `projectionParallaxViewerOffsetY`、壁面からの距離は `projectionParallaxViewerDistance`、投影面余白は `projectionViewportMargin`、消失点は `projectionParallaxVanishingPointX/Y` で管理画面から調整する。消失点は `0,0` を画面中央、Xは `-1〜1` を左右端、Yは `-1` を上端、`1` を下端、`5` をさらに下方として扱う。
- パースは消失点だけでは決まらない。`projectionParallaxViewerDistance` を焦点距離の倍率として使い、壁面に近いほどパースが強く、遠いほど穏やかになるようにする。`projectionParallaxViewerOffsetX/Y` は基準カメラ位置の左右/上下ずれとして投影式に加える。
- 壁投影モードでは、鑑賞方向は画面正対ではなく消失点方向に固定する。消失点は投影の固定アンカーとして扱い、笹舟/カメラの左右・前後移動は通常通り再投影差として短冊と笹に反映する。奥行基準順は短冊Z間隔の基準であり、基準短冊の画面位置を固定するための補正には使わない。
- 消失点マーカーは `projectionParallaxMarkerEnabled` でON/OFFする。余白ガイドラインの表示はマーカーON/OFFに連動させる。
- 遠景以外の視差対象は、投影面余白の内側を基準に配置する。ただし視差による回り込みは、余白部分にはみ出して描画してよい。
- 視差は単純なスライドではなく、各短冊・笹を仮想Z距離に置き、笹舟で示す鑑賞ポイント移動後に焦点距離と消失点を使って再投影した差分をCSS変数へ渡す。`projectionParallaxPopoutStrength` はレイヤー自体の押し出し量ではなく、笹舟/カメラの前後または上下移動倍率として使う。短冊のZ距離は基準短冊からの差分に `projectionParallaxDepthMultiplier` を掛けて調整し、`0` では全短冊を同一深度、`1` では標準間隔、`1` 超では前後差を広げる。
- `camera` モードでは、仮想カメラZ移動で投影面そのものが画角内で拡大縮小するため、投影面Zのスケールを基準にして全体を補正する。これにより投影面上の固定要素は画面内で極端にズームせず、手前/奥の短冊と笹だけが投影面に対する相対視差として動く。
- 投影画面下部の笹舟は、カメラ移動量のモニターとして常時見えるように扱う。コンテナは固定し、移動量0の位置には小さな星を置き、笹舟グループだけを `--viewer-oracle-x/y` で動かす。水面反射のような演出は使わず、笹舟本体が見えることを優先する。
- 建物投影では地表の鑑賞ポイントが投影面より下になるため、消失点Yは画面下端を超えて `5` まで設定できる。消失点マーカー自体は画面外へ出る場合がある。
- 消失点マーカーON時は、同じ再投影計算を使うワイヤーフレーム3DボックスをSVGで描く。ボックスは短冊・笹と同様に笹舟のカメラ移動へ連動して変形する。
- 消失点マーカーと3Dボックスは調整用オーバーレイなので、短冊より手前に表示する。短冊との干渉を避けるために背面へ下げると確認性が落ちるため、チラつき対策はレイヤー順ではなく描画安定化で行う。
- 視差表示のチラつきは、表示直後ではなく数秒後から目立つ場合がある。これは仮想カメラ移動が大きくなるにつれて、短冊や3Dボックスの座標がサブピクセル単位で毎フレーム変わり、アンチエイリアスや影の再ラスタライズが揺れることが主因になりやすい。
- 3Dボックスは確認できればよい補助表示なので、描画負荷を優先して簡素に保つ。現在は前面矩形、背面矩形、奥行き接続線の3本のSVG `path` だけで描き、面、頂点、太いハロー、消失点補助線は描かない。
- 3DボックスのSVG要素は毎フレーム作り直さない。初回だけ `path` を生成し、以後は `d` 属性だけ更新する。`replaceChildren()` を毎フレーム呼ぶと、線や頂点の再生成でチラつきやすい。
- 3Dボックスの座標と短冊の視差オフセットは `0.5px` 単位に量子化する。ピクセル未満の連続更新は投影環境で明滅に見えやすい。確認用オーバーレイは `PARALLAX_OVERLAY_FRAME_MS = 1000 / 15` 程度に更新を間引く。
- 月メニュー表示中は3Dボックスの更新を止め、静止表示のままにする。メニューを閉じた次フレームで再同期し、メニュー操作中の再合成負荷を避ける。
- 3Dボックスの大きなSVG全体や頂点へ `drop-shadow` を付けると、カメラ移動中に影の再ラスタライズが目立つ。動的なSVGには広い範囲のフィルタ、発光、破線を避け、細い単色実線だけで表示する。
- 短冊は独立した `scale` / `translate` プロパティと `transform` アニメーションを混在させない。視差移動、揺れ、奥行きスケールは `transform` に統合し、CSS変数の更新も値が変わった時だけ行う。短冊の視差移動は `0.5px`、視差スケールは `0.005` 単位に丸める。短冊には `backface-visibility: hidden`、`contain: layout paint`、`will-change: transform, opacity` を付け、合成を安定させる。
- 突風演出は左右方向の掃引に加えて短冊の疑似奥行きも使い、奥の短冊ほど少し遅れて揺れ始めるようにする。
- 視差ON時は、投影画面下部に小さな七夕モチーフの隠しインジケータを出し、仮想鑑賞ポイントの左右・前後移動を確認できるようにする。
- 3D星空ドームは視覚効果が薄いためキャンセルする。WebGLを使う場合は、通常背景ではなく特別演出中の歪み、光の橋、または短時間の空間演出に絞って再検討する。
- 流星群演出ではWebGLレイヤーを短時間だけ有効化し、天の川の星々がベガとアルタイルの間へ集まって橋を形成し、完成時に強く発光する。星の集合はイージングと分散開始タイミングを持ち、橋の手前で減速し、一部は目標地点を少し越えて戻る。橋は形成後に約2秒保持し、その後ゆっくり散開しながら渦巻き、流星群終盤まで徐々にフェードアウトする。

## 開発つまづきポイント

今回の大きな改修では、見た目の違和感が単一のバグではなく、投影幾何、DOM合成、CSSアニメーション、SSE/ポーリング更新、端末フォント差の重なりとして出た。次回同種の開発では、次の点を先に確認する。

- 視差3Dの奥行きには、短冊の画面上Y位置を混ぜない。Yは投影面上の高さであり、鑑賞方向が正対または消失点固定なら、奥行きはz順だけから作る。
- 消失点だけではパース量は決まらない。鑑賞地点の左右、上下、壁面からの距離を別パラメータとして持たせ、UIにもXYZとして出す。
- 奥行基準順は短冊同士のZ間隔の基準であり、基準短冊を画面上に固定する補正ではない。基準固定を入れると、笹舟/カメラ移動が見えなくなる。
- 「実カメラ」モードでは、仮想カメラのZ移動により現実カメラ画角内の投影面がズームイン/アウトして見える。投影面Zのスケールを基準に補正しないと、短冊以外の平面も不自然に拡大縮小する。
- 3Dボックスは短冊と同じ見た目補正をそのまま使うと変化が分かりにくい。確認用表示として、専用の再投影、少し強めのカメラ移動、広めの奥行きで描く。
- 消失点マーカーや3Dボックスを短冊の後ろへ下げると確認性が落ちる。チラつき対策はレイヤー順ではなく、更新頻度、量子化、フィルタ削減、DOM再生成回避で行う。
- SVGオーバーレイは毎フレーム要素を作り直さない。初回に `path` を作り、以後は `d` 属性だけ更新する。`replaceChildren()` はチラつきと負荷の原因になりやすい。
- 短冊の `transform` は一箇所に集約する。独立した `scale` / `translate` と `transform` アニメーションを混在させると、突風、視差、入退場の境目で微動ワープしやすい。
- 短冊ローテーション中にz-indexやDOM順を即時変更すると、既存短冊が奥へ微動ワープして見える。奥の対象短冊を先に退場させ、残り短冊のZを短時間でなだらかに更新し、その後に新規短冊を入れる。
- 入れ替え対象は最奥から指定枚数にする。表示中の順序や作成順ではなく、投影上の奥行き順で選ぶ。
- `renderPosition` をポーリングやSSEのたびに再計算すると、保存済み配置でも短冊が微動する。明示的な位置変更、表示数増加、新規表示など、必要なときだけ位置探索を走らせる。
- 投影画面のポーリングは差分シグネチャを必ず見る。差分なしで短冊、雲、3D視差を再描画すると、15秒前後の点滅や雲アニメーションのリセットとして見える。
- 月メニューや隠しUIは、3D演出の高負荷時に合成競合でチラつきやすい。メニュー表示中は3Dボックス更新を止め、操作UIの合成負荷を下げる。
- 3DボックスなどSVGで描く調整対象をドラッグする場合、SVG内のcircle/pathを直接ハンドルにすると、再投影更新やpointer captureの相性でドラッグが細切れになりやすい。短冊ドラッグと同じように、安定したHTML要素をハンドルにし、`window` の `pointermove` で移動量を追うほうが安定する。
- 座標調整UIを追加した場合は、通常CSSだけでなくスマホ向けメディアクエリの後勝ち上書きも確認する。笹左Xは通常指定では効いていたが、スマホ幅の `.bamboo-left` が `left` を固定していたため、現場端末では効かないように見えた。
- Android端末ではブラウザ表示領域とOS UIの安全領域がズレる場合がある。画面下端に置く笹舟などは、サイズ連動だけでなく下端オフセットを現場で調整できるようにする。
- 縦書きフォントは端末内蔵フォントに依存しすぎない。特にAndroidではゴシック/丸ゴシック系の縦書き対応が弱い端末があるため、同梱フォントを選択時に読み込める構成にする。
- カラー絵文字フォントはファイルサイズ、描画負荷、ブラウザ差が大きい。通常機能ではなく実験スイッチとして扱い、選択時だけ読み込む。
- 多数短冊時の処理落ちは、DOMを仮想化して枚数を消すより、投影品質を保ちながら軽く描くほうが向いている。次回は表示枚数に応じた軽量モード、影/フィルタ削減、視差更新の間引きを優先する。

## 次回修正案

- 管理画面の雲スライダーと消失点スライダーは、画面幅が広い時にグループボックスからはみ出ることがある。横長レイアウトでも収まるように、グループ内の最小幅、折り返し、スライダー長の上限を見直す。
- 演出ONとプリセット1〜3の呼び出しは、外部アプリからOSC経由でトリガーできるようにする。管理画面のボタン操作と同じ内部APIへつなぐ形にするとよい。
- 笹舟は自動移動モードとOSCモードを切り替えられるようにする。OSCモードではヘッドトラッキングを想定し、鑑賞地点パラメータをOSC経由でXYZリアルタイム更新して、笹舟の位置に反映する。

## 縦書きレイアウト

- 投稿・投影とも `.tanzaku-text` を共通利用する。
- テキストは縦書き `writing-mode: vertical-rl`。
- 文字は短冊内で左右中央、上揃え。
- 1行でも2行でも中央線が揃うように、`left: 50%` と `translateX(-50%)` で物理的に中央へ置く。
- `text-align: start` にして、縦書きの開始位置を上側に固定する。
- 短冊の長さは最大13文字が1列で収まることを基準にしている。
- こより穴と文字の距離を確保するため、上余白は `--tanzaku-padding-top` で管理する。

## 星空・雲・月・流れ星

- 背景には星空、流れる雲、月、流れ星をCSSで描画している。天の川、強化星空、ベガ、アルタイルはCanvas 2Dレイヤーで描画する。
- 通常時の重なり順は、星、月、雲、笹、短冊の順に手前へ重ねる。月メニューだけは操作UIとして最前面に出す。
- 雲は月の上を通るため、月が隠れたときに朧月に見えるよう、発光と半透明レイヤーの重なりを使っている。
- 雲は管理画面の `projectionCloudCount`、`projectionCloudOriginY`、`projectionCloudSeed` から生成する。雲はX位置を調整対象にせず、常に左の画面外から発生して右の画面外で消える。高さだけを画面全体の相対値で扱い、中央を `Y=0`、上端を `Y=-1`、下端を `Y=1` とする。既定は中央の `Y=0`。シード値は文字列で、同じシードなら同じランダム配置になる。
- 古いChromeでは、`radial-gradient`、`filter`、`box-shadow`、半透明レイヤーの合成により、星や月など一部の背景表現だけが表示されないことがある。投影端末はChrome最新版を推奨する。
- 流れ星の発生頻度は控えめにし、周期は `80s`。
- 流れ星の表示時間だけを短くして、発生頻度を上げずに速度感を調整している。

## 突風演出

- 流れ星の少し前に、短冊へ風が吹き抜ける演出を入れている。
- 突風は `METEOR_DELAYS_MS` と `WIND_LEAD_MS` で流れ星タイミングに同期する。
- 各短冊は `x` 座標順に遅れて揺れ始める。
- 風の吹き抜け速度は `WIND_SWEEP_MS` で調整する。
- 短冊の揺れはこより先端を基準にした振り子表現にしている。
- かくつき対策として、小刻みな往復ではなく「大きく一度傾いて滑らかに戻る」動きにしている。
- 通常時の短冊にも `transform-origin: 50% -18vh` を指定し、突風開始時に支点が変わって跳ねないようにしている。

## データと公開時の注意

- `data/wishes.json` は利用者投稿を含むためGit管理しない。
- `data/.gitkeep` だけをコミットし、初回起動時にサーバーが `data/wishes.json` を生成する。
- 公開サーバーで運用する場合は、`ADMIN_KEY` を必ず変更する。
- 公開運用ではHTTPS、永続ストレージ、バックアップ、アクセス制限、ログ管理を別途検討する。
- 現在の実装は試作開発版であり、攻撃、荒らし、イタズラ投稿、大量投稿、管理画面への不正アクセスへの十分な防御は備えていない。
- インターネットへ直接公開する場合は、管理画面のアクセス制限、投稿頻度制限、CAPTCHA、WAF、監視、通報・削除フローなどを別途設計する。
- 多人数・長期公開運用では、ファイル保存の代わりにDB、権限分離、監査ログ、世代管理されたバックアップを検討する。

## 主な調整値

- 投稿行数: `MAX_WISH_LINES = 2`
- 1行文字数: `MAX_WISH_LINE_LENGTH = 13`
- 総スロット数: 既定 `15`、管理画面の投影設定で変更可能
- 表示短冊数: 既定 `12`、管理画面の投影設定で変更可能。最大値は総スロット数
- ローテーション入替数: 既定 `1`、管理画面の投影設定で変更可能。最大値は表示短冊数。管理画面では表示数と入替数を増減ボタンで調整する
- 文字アニメーション速度: 既定 `240ms`、管理画面の投影設定で変更可能。最速 `60ms`
- 短冊ローテーション間隔: 既定 `18000ms`、管理画面の投影設定で変更可能
- 自動演出: 既定 `OFF`、管理画面の投影設定で変更可能
- イベント間隔: 既定 `300000ms`、管理画面の投影設定で変更可能
- 天の川演出: 明るさは既定 `1.75`、粒子数は既定 `340`、強い瞬き割合は既定 `0.045`、きらめき、強い瞬き、変化速度、瞬き周期分散、瞬き強度分散は既定 `1`。管理画面の投影設定で変更可能
- 雲の発生量: 既定 `3`、管理画面の投影設定で変更可能
- 雲の高さ: 既定 `Y=0`、管理画面の投影設定で変更可能
- 雲シード: 既定 `tanabata-clouds`、文字列として管理画面の投影設定で変更可能
- 視差立体感: 既定 `OFF`、管理画面の投影設定で変更可能
- 笹舟移動モード: 既定 `mapping`、管理画面の投影設定で変更可能
- 左右移動倍率: 既定 `1`、管理画面の投影設定で変更可能
- 前後/上下移動倍率: 既定 `0`、管理画面の投影設定で変更可能
- 鑑賞地点左右オフセット: 既定 `0`、管理画面の投影設定で変更可能
- 鑑賞地点上下オフセット: 既定 `0`、管理画面の投影設定で変更可能
- 壁面からの距離: 既定 `2.5`、管理画面の投影設定で変更可能
- 投影面余白: 既定 `0`、管理画面の投影設定で変更可能
- 視差消失点: 既定 `0,0`、管理画面の投影設定で変更可能
- 消失点マーカー: 既定 `OFF`、管理画面の投影設定で変更可能
- 投影設定プリセット: 3枠。総スロット数、表示短冊数、ローテーション入替数、文字アニメーション速度、短冊ローテーション間隔、自動演出、イベント間隔、天の川、雲、視差設定、笹舟移動モード、消失点マーカー、左右/前後移動倍率、投影面余白を保存する
- 流れ星周期: CSS `.meteor` の `animation: meteor-fall 80s linear infinite`
- 突風先行時間: `WIND_LEAD_MS = 3200`
- 突風吹き抜け時間: `WIND_SWEEP_MS = 2200`
- 短冊揺れ時間: `WIND_GUST_MS = 2200`
