# Development Notes

このドキュメントは、試作開発中に調整した判断や今後の開発で参照したい要点をまとめたものです。

## 全体方針

- 大学構内WiFiからのローカル運用を前提に、Node.js標準ライブラリだけで構成している。
- 外部DBは使わず、願い事は `data/wishes.json` に保存する。
- 管理者が承認した願い事だけを投影画面に表示する。
- 投影画面はプロジェクションマッピング用途を優先し、暗い背景、強い発色、縦書き短冊、星空・雲・月・流れ星の演出で構成する。

## 投稿入力

- 願い事は最大2行。
- 1行ごとにテキストボックスを分ける。
- 改行コードは無視し、貼り付け時も1行内に収める。
- 1行の最大文字数は13文字。
- Enterキーで投稿されないように、入力欄の `keydown` で送信を抑止している。
- 投稿画面のプレビューは投影画面と同じ `.tanzaku-text` を使い、文字配置の差が出にくいようにしている。

## 管理画面

- 管理キーは `ADMIN_KEY` 環境変数で指定する。未指定時は試作用の `tanabata2026`。
- 新規投稿や承認状態の変更は Server-Sent Events で管理画面へ通知する。
- SSEが不安定な場合に備え、30秒間隔のポーリングも残している。
- 管理キーはブラウザの `localStorage` に保存しているため、共有PCや公開端末では注意する。
- 投影設定フォームは、入力中にSSEや30秒ポーリングの `refresh()` が走っても未適用の値をサーバー値で上書きしない。適用成功時だけ確定値で再描画する。

## 短冊表示

- 総スロット数、表示数、入れ替え数は管理画面から調整できる。既定は総スロット15個、表示12枚。
- 表示数は総スロット数を超えない。総スロット数を下げて表示数の上限が下がる場合は、表示数も上限内に調整する。
- 入れ替え数は表示数を超えない。表示数を下げて入れ替え数の上限が下がる場合は、入れ替え数も上限内に調整する。
- 待機スロット数は `総スロット数 - 表示数` で決まる。
- 投影画面は `/api/approved` を7秒間隔で取得し、新規承認された願い事を検出する。
- 新規承認された願い事は通常ローテーションを待たず、表示中の一番古い短冊を待機へ戻して優先表示する。
- 通常ローテーションでは、表示中の一番古い短冊を待機へ戻し、待機中の一番古い短冊を表示へ上げる。
- 各スロットは `rotationOrder` を持ち、表示または待機に入った順番で古いものから入れ替える。
- 新規表示された短冊は一番新しい表示扱いになるため、次のローテーションですぐ消えにくい。
- 新規承認で追加された短冊は、既存短冊より前面に表示する。
- 投影端末に保存済みレイアウトがないスロットでは、新規短冊の表示時に候補位置から既存短冊との重なりが少ない場所を選ぶ。
- 表示OKな願い事数が総スロット数より少ない場合でも、同じ願い事が表示中に複数出ないようにする。待機スロットが空なら、古い表示短冊を別スロットへ順次移して見せる。
- 入れ替え数は表示数以下に制限する。
- ローテーション中と新規取得が重なる稀なケースは、運用上の影響が小さいため許容する。
- 待機中のメッセージも保持し、表示数を超えた願い事はページングではなく周期的な入れ替えで見せる。
- 新規表示は一文字ずつ出るタイプ風演出。
- タイプ風演出は文字数に応じて速度を変え、短いメッセージほど加速度的に遅くする。
- 消える短冊は風に舞い上がって天に昇る演出。
- 表示中の短冊はドラッグで位置調整でき、位置は `localStorage` に保存される。

## 投影演出

- 管理画面の「流星群」ボタンから、投影画面へ短時間の演出リクエストを送る。
- 管理画面の投影設定で自動演出をONにすると、指定したイベント間隔で流星群演出を自動発生させる。
- 自動演出は既定OFF、イベント間隔の既定は `300000ms`。
- 演出リクエストはサーバーメモリ上の一時状態で、30秒で失効する。投稿データや設定ファイルには保存しない。
- 投影画面は演出リクエストを2.5秒間隔で確認する。
- 流星群レイヤーは月、笹、短冊より後ろに表示する。
- 流星群演出中は通常ローテーションと新規短冊の導入を一時停止する。
- 流星群の発生後、少し待ってから表示中の短冊を古い順に退場させる。
- 全短冊が退場した後、流星群をフェードアウトし、承認済み一覧を再反映して通常表示へ戻す。

## 縦書きレイアウト

- 投稿・投影とも `.tanzaku-text` を共通利用する。
- テキストは縦書き `writing-mode: vertical-rl`。
- 文字は短冊内で左右中央、上揃え。
- 1行でも2行でも中央線が揃うように、`left: 50%` と `translateX(-50%)` で物理的に中央へ置く。
- `text-align: start` にして、縦書きの開始位置を上側に固定する。
- 短冊の長さは最大13文字が1列で収まることを基準にしている。
- こより穴と文字の距離を確保するため、上余白は `--tanzaku-padding-top` で管理する。

## 星空・雲・月・流れ星

- 背景には星空、流れる雲、月、流れ星をCSSで描画している。
- 雲は月の上を通るため、月が隠れたときに朧月に見えるよう、発光と半透明レイヤーの重なりを使っている。
- 古いChromeでは、`radial-gradient`、`filter`、`box-shadow`、半透明レイヤーの合成により、星や月など一部の背景表現だけが表示されないことがある。投影端末はChrome最新版を推奨する。
- 流れ星の発生頻度は控えめにし、周期は `80s`。
- 流れ星の表示時間だけを短くして、発生頻度を上げずに速度感を調整している。

## 突風演出

- 流れ星の少し前に、短冊へ風が吹き抜ける演出を入れている。
- 突風は `METEOR_DELAYS_MS` と `WIND_LEAD_MS` で流れ星タイミングに同期する。
- 各短冊は `x` 座標順に遅れて揺れ始める。
- 風の吹き抜け速度は `WIND_SWEEP_MS` で調整する。
- 短冊の揺れはこより先端を基準にした振り子表現にしている。
- かくつき対策として、小刻みな往復ではなく「大きく一度傾いて滑らかに戻る」動きにしている。
- 通常時の短冊にも `transform-origin: 50% -18vh` を指定し、突風開始時に支点が変わって跳ねないようにしている。

## データと公開時の注意

- `data/wishes.json` は利用者投稿を含むためGit管理しない。
- `data/.gitkeep` だけをコミットし、初回起動時にサーバーが `data/wishes.json` を生成する。
- 公開サーバーで運用する場合は、`ADMIN_KEY` を必ず変更する。
- 公開運用ではHTTPS、永続ストレージ、バックアップ、アクセス制限、ログ管理を別途検討する。
- 現在の実装は試作開発版であり、多人数・長期公開運用ではファイル保存の代わりにDBを検討する。

## 主な調整値

- 投稿行数: `MAX_WISH_LINES = 2`
- 1行文字数: `MAX_WISH_LINE_LENGTH = 13`
- 総スロット数: 既定 `15`、管理画面の投影設定で変更可能
- 表示短冊数: 既定 `12`、管理画面の投影設定で変更可能。最大値は総スロット数
- ローテーション移動量: 既定 `1`、管理画面の投影設定で変更可能。最大値は表示短冊数
- 文字アニメーション速度: 既定 `240ms`、管理画面の投影設定で変更可能。最速 `60ms`
- 短冊ローテーション間隔: 既定 `18000ms`、管理画面の投影設定で変更可能
- 自動演出: 既定 `OFF`、管理画面の投影設定で変更可能
- イベント間隔: 既定 `300000ms`、管理画面の投影設定で変更可能
- 流れ星周期: CSS `.meteor` の `animation: meteor-fall 80s linear infinite`
- 突風先行時間: `WIND_LEAD_MS = 3200`
- 突風吹き抜け時間: `WIND_SWEEP_MS = 2200`
- 短冊揺れ時間: `WIND_GUST_MS = 2200`
