# Tanabata Fes 2026

大学構内WiFiからアクセスするローカル運用向けの七夕願い事投稿・承認・投影Webアプリです。

利用者が投稿した願い事を管理者が確認し、承認されたものだけを縦書き短冊として投影画面に表示します。プロジェクションマッピング用途を想定して、星空、雲、月、流れ星、短冊のローテーション演出を含みます。

## 機能

- 願い事投稿フォーム
- 2行まで、1行13文字までの入力制限
- 投稿画面での短冊プレビュー
- 管理画面での承認、保留、非表示、削除
- 手動確認、自動承認、AI自動モデレートの切り替え
- 管理画面の自動更新
- 管理画面で投稿詳細表示、投影設定、演出を操作
- 投影画面での縦書き短冊表示
- 表示数、移動量、文字アニメーション速度、ローテーション間隔の調整
- 新規承認短冊の優先表示と、古い表示からの順次ローテーション
- 文字数に応じたタイプ風描画
- 消える短冊の舞い上がり演出
- 流れ星前の突風による短冊の振り子揺れ
- 手動または自動の流星群演出
- 短冊位置のドラッグ調整と保存

## 画面

- 投稿: `http://localhost:3000/`
- 管理: `http://localhost:3000/admin`
- 投影: `http://localhost:3000/projection`

## 必要環境

- Node.js 20以上

外部npmパッケージは使っていません。

## 起動方法

本番相当の起動は `.env` を作って必要な値を書き、`npm start` を実行します。雛形は [`.env.example`](.env.example) です。

```sh
npm start
```

開発を止めずに続けるときは `npm run dev` を使います。Caddy が向いている運用中Nodeの反対側のポートで起動するので、運用中の `3000` と運用中Nodeにぶつかりません。初回は `3001` が運用中なら `3002`、切り替え後は `3002` が運用中なので `3001` を開発用に使います。

```sh
npm run dev
npm run smoke:dev
```

テスト投稿や承認操作を本番データに混ぜずに確認する場合は、隔離データで起動します。

```sh
npm run dev:isolated
npm run smoke:dev
```

`HOST`、`PORT`、`ADMIN_KEY`、`OPENAI_API_KEY` などは `.env` に書きます。管理キーを指定しない場合は `tanabata2026` です。

AI自動モデレートを使う場合は、`OPENAI_API_KEY` を `.env` に書きます。ブラウザや設定ファイルには保存しません。AIモードは安全寄りに動作し、連絡先、SNS ID、URL、個人情報らしき内容、名指し攻撃、強い表現、`しね`、`タヒね`、`し ね` のような検索避け、悪戯や下品な表現の可能性がある投稿は自動承認せず手動確認待ちにします。

## データ保存

願い事は `data/wishes.json` に保存されます。このファイルは利用者投稿を含むためGit管理対象外です。存在しない場合はサーバー起動時に自動作成されます。

## 運用の流れ

1. 利用者が投稿画面から願い事を送信します。
2. 手動確認モードでは `pending`、自動承認モードでは `approved`、AI自動モデレートでは判定結果に応じて `approved`、`pending`、`rejected` として保存されます。
3. 管理者が管理画面で承認すると `approved` になり、投影画面に表示されます。
4. 新規承認された願い事は優先表示されます。
5. 表示数を超える願い事は投影画面側で古い表示から一定時間ごとにローテーションします。
6. 投影の表示数、移動量、文字アニメーション速度、ローテーション間隔、自動演出は管理画面から調整できます。

投影画面はプロジェクションマッピング用途を想定し、暗い背景、強めの発色、縦書き短冊で構成しています。

## 保守運用

運用中の確認には次のコマンドを使います。

```sh
npm run check
npm run backup
npm run smoke:prod
```

`.env` を変えた場合は Node プロセスを再起動します。静的ファイルの変更は再起動不要です。`server.js` を変更する場合は、`npm run dev` で反対側ポートに開発用サーバーを起動し、`npm run smoke:dev` で確認してから切り替えます。詳しくは [運用手順](docs/OPERATIONS.md) を参照してください。

Caddyを使う場合は、Nodeの `PORT` を `.env` で 3001 / 3002 に切り替え、Caddyの向き先だけを `npm run proxy:switch` で変えます。
