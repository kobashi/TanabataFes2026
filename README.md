# Tanabata Fes 2026

大学構内WiFiからアクセスするローカル運用向けの七夕願い事投稿・承認・投影Webアプリです。

利用者が投稿した願い事を管理者が確認し、承認されたものだけを縦書き短冊として投影画面に表示します。プロジェクションマッピング用途を想定して、星空、雲、月、流れ星、短冊のローテーション演出を含みます。

## 機能

- 願い事投稿フォーム
- 2行まで、1行13文字までの入力制限
- 投稿画面での短冊プレビュー
- 管理画面での承認、保留、非表示、削除
- 管理画面の自動更新
- 投影画面での縦書き短冊表示
- 表示12枚、待機3枚による周期的な入れ替え
- 新規表示時のタイプ風描画
- 消える短冊の舞い上がり演出
- 流れ星前の突風による短冊の振り子揺れ
- 短冊位置のドラッグ調整と保存

## 画面

- 投稿: `http://localhost:3000/`
- 管理: `http://localhost:3000/admin`
- 投影: `http://localhost:3000/projection`

## 必要環境

- Node.js 20以上

外部npmパッケージは使っていません。

## 起動方法

```sh
npm start
```

標準では `0.0.0.0:3000` で待ち受けます。同じWiFi内の端末からは、サーバーPCのIPアドレスを使ってアクセスします。

```sh
HOST=0.0.0.0 PORT=3000 ADMIN_KEY=任意の管理キー npm start
```

管理キーを指定しない場合は `tanabata2026` です。実運用では `ADMIN_KEY` を指定してください。

## データ保存

願い事は `data/wishes.json` に保存されます。このファイルは利用者投稿を含むためGit管理対象外です。存在しない場合はサーバー起動時に自動作成されます。

## 運用の流れ

1. 利用者が投稿画面から願い事を送信します。
2. 願い事は `data/wishes.json` に `pending` として保存されます。
3. 管理者が管理画面で承認すると `approved` になり、投影画面に表示されます。
4. 表示数を超える願い事は投影画面側で一定時間ごとにローテーションします。

投影画面はプロジェクションマッピング用途を想定し、暗い背景、強めの発色、縦書き短冊で構成しています。
